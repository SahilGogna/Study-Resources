# Prompt Library

# Phase 1: Planning & Strategy

# Job Description Skill Extractor

I'm applying for a [Role Title] position. Here's the job description:

[PASTE JOB DESCRIPTION]

Please:
1. Extract all required skills, tools, and competencies mentioned
2. Extract desired/nice-to-have skills
3. Create a table with three columns:
   - Must-Have Skills (I need these to be competitive)
   - Good-to-Have Skills (I should learn these in 60-90 days)
   - Nice-to-Have Skills (These are long-term bets, 6-12 months)
4. For each skill, estimate learning time if I'm starting from scratch
5. Highlight the top 3 skills that appear most frequently across similar roles
6. Suggest one portfolio project that would demonstrate mastery of the top 3 skills

Format the output as a clear table + bullet points for recommendations.

# Target Role + Company Deep-Dive

I'm targeting:
- Role: [Data Engineer / SWE / Product Manager / etc.]
- Focus industries: [e.g., Fintech, Healthtech, Climate]
- Target companies: [e.g., Shopify, Wealthsimple, TD]
- Current location: [City, Province]

Please provide:
1. Current market demand for this role in Canada (estimate of open roles)
2. Typical salary range in [your city] for this role (entry, mid, senior levels)
3. Top 3 skills that differentiate HIGH-paying roles from average roles
4. 5 companies in [your city] actively hiring for this role
5. Key competitors' requirements (what do top 3 companies emphasize?)
6. Most common interview formats (process, rounds, timeline)
7. 3 projects a candidate could build to stand out for this role

Format: Use bullet points and tables where helpful.

Role: [Your target role]

# Skills Gap + Learning Plan

My current skills:
[List your skills and years of experience]

Target role: [Role Title]
Target companies: [e.g., Shopify, Amazon, TD]
Timeline to apply: [8 weeks / 12 weeks / 16 weeks]

Create a prioritized learning plan:
1. Rank the top 10 skills I need for [role] by ROI (return on investment)
   - Factor in: demand in job postings, gap between my skill + requirement, learning time
2. For each of the top 5 skills:
   - Estimated time to "good enough" level (60-90 days)
   - Free resources to learn (courses, tutorials, documentation)
   - Hands-on projects to solidify learning
   - How to add this skill to my resume + portfolio
3. Create a week-by-week breakdown for the next [12 weeks]
   - What skill to focus on each week
   - How much time to spend (hrs/week)
   - One small project or milestone per week
4. Red flags: If I'm not making progress by [week X], what should I pivot to?

Format: Use a table for skills ranking + a calendar/timeline for the learning plan.

# Phase 2: Resume & Portfolio

# Resume Bullet Optimizer

I'm applying for: [Role Title] at [Company]
Here's my current resume bullet:
"[CURRENT BULLET]"

Please rewrite this bullet to:
1. Lead with the impact/outcome, not the task
2. Include 1-2 quantifiable metrics (% improvement, time saved, revenue generated, users impacted, etc.)
3. Show the technical skill relevant to [role]
4. Match keywords from the job posting
5. Be specific and memorable (not generic)

Provide:
- Revised bullet (1-2 lines)
- Explanation of why this is stronger
- 2-3 variations if the original context doesn't support the metrics I suggested

Format: Original → Revised + notes.

# Resume Tailoring for Specific Role

My background: [Brief summary: e.g., "SWE with 3 years backend experience, Python, AWS, shipped 2 projects to production"]
Target role: [e.g., Senior Backend SWE at [Company]]
Target company: [Company name]
Job description focus areas: [Copy the top 3-4 requirements from the posting]

Analyze my resume:
1. Which of my skills are MOST relevant to this role? (Rank top 5)
2. Which skills are nice-to-have but less critical? (Rank)
3. For each of the top 3 requirements from the job posting, do I have evidence on my resume?
4. What's missing or weak?
5. What should I move to the top of my resume?

Provide:
- Reordered sections (what goes where for max impact)
- 1-2 key bullets I should re-write for this specific role
- Skills to emphasize + skills to de-emphasize
- One thing I'm missing entirely (if so, can I show this via portfolio / LinkedIn?)

Format: Concise recommendations + marked-up resume example.

# Portfolio Case Study Generator

Project: [Your project name / GitHub repo]
Role you're targeting: [e.g., Data Engineer, SWE, etc.]
Your role in the project: [Solo / Lead / Contributor]
Tech stack: [e.g., Python, Spark, AWS S3, Airflow]
What it does: [1-sentence description]
Key metrics / outcomes: [e.g., "Reduced data load time by 60%", "Processed 10M records/day", etc.]

Create a professional 1-page case study:
1. Executive summary (2-3 lines): What problem did you solve?
2. Background (3-4 lines): Why was this project valuable? Business context?
3. Approach (5-6 lines): How did you think about it? What choices did you make? Why?
4. Results (3-4 lines): What happened? Metrics? Impact?
5. Key learnings (3-4 lines): What surprised you? What would you do differently?
6. Technical highlights (3-5 bullets): Specific technical decisions or challenges you overcame
7. [Optional] What's next: If you extended this project, what would you build?

Also provide:
- 1-2 demo video scripts (2-3 mins each) – what to show, what to say
- 3 ways to present this project in an interview without reading the case study
- Metrics you can use on your resume based on this project

Format: Professional 1-pager + video scripts + resume bullets.

# Phase 3: Networking & Outreach

# Cold LinkedIn DM Template Generator

I want to reach out to: [Name, Title, Company]
Their recent post / article: [Topic or quote from their post]
My situation: [e.g., "Transitioning into cloud engineering, 6 years SWE background, based in Toronto"]
My ask: [e.g., "15 min coffee chat about their experience transitioning to cloud", "Advice on landing first PM role", etc.]
My specific question: [e.g., "What surprised you most about cloud engineering at Shopify?", "How did you prepare for your PM interviews?", etc.]

Create 3 versions of a LinkedIn DM:
1. Warm opening (genuine, not generic)
   - Reference their post specifically (1-2 specific points from their post)
   - Show your genuine interest (not "just fishing")
   - Mention any mutual connection or shared experience if applicable
2. Your situation (brief, relevant)
3. Specific ask (not "can we talk?" but "I'm curious about X because Y")

Each version should be:
- Under 150 words
- Personalized (not copy-paste)
- Respectful of their time
- Include a clear ask (coffee chat? email exchange? etc.)

Then provide:
- Follow-up message (if they don't respond in 1 week)
- How to handle if they say "no" or don't respond (stay positive, stay visible)

Format: 3 versions + follow-up + handling rejections.

# Coffee Chat Prep Guide

I have a coffee chat scheduled with: [Name, Title, Company, Role]
Their background: [What do you know about them?]
Your target role: [Your aspiration]
Your goal from this chat: [e.g., "Understand what it takes to land this role", "Get a warm referral", "Learn about the company culture"]

Create a prep guide:
1. Opening (how to start the call, build rapport)
2. 5-7 thoughtful questions to ask:
   - Questions about their role + what they love/hate about it
   - Questions about their interview experience
   - Questions about the company culture / team dynamics
   - If relevant: question about hiring / if they know of open roles
   - What surprised them most in their first year?
3. What NOT to do (don't: ask them to refer you immediately, don't dump your resume, etc.)
4. How to end (thank them, ask if you can follow up, ask for 1 more intro if appropriate)
5. After the call: 1-day follow-up message (thank you + specific insight you valued)
6. How to stay in touch without being annoying (once every 3-6 months? Share relevant article?)

Then provide:
- Email template for 1-day follow-up
- Email template for staying in touch (3 months later)
- How to ask for a warm referral without being pushy

Format: Prep guide + templates + timing guidance.

# LinkedIn Profile Optimizer

Current headline: [Your current headline]
Current "About" section: [Your current about section, or describe what's there]
Your target role: [e.g., Data Scientist, Cloud Engineer, PM]
Target companies: [e.g., Shopify, Wealthsimple, Amazon]
Location: [City, Province]
Key differentiator: [What makes you unique? E.g., "Fintech background + ML", "Cloud migration expert"]

Optimize:
1. Headline: Rewrite to include:
   - Role + key skills
   - Specific focus area or industry
   - Call-to-action or openness signal
   - Max 220 characters

2. About section (rewrite, aim for 100-150 words):
   - Who you are (role + experience level)
   - What you do / build (be specific, not generic)
   - What you're looking for (role, industry, focus)
   - What you want to connect about (vs just accepting all requests)
   - A light call-to-action ("Let's chat about X")
   - Remove: clichés ("passionate", "driven"), buzzwords, generic statements
   - Add: personality, specificity, business focus

3. Skills section:
   - Top 10 skills ranked by relevance to target role
   - Order them strategically (most job-posted skills first)
   - Ensure they match keywords from target job postings

4. Recommendations section:
   - What kind of recommendations should you request?
   - 1-line "ask" to suggest to people who might recommend you (vs generic "recommend me" requests)

5. Activity:
   - What should you post about to attract recruiters + people in your target companies?
   - 5 post ideas (specific, not generic)

Format: Revised sections + reasoning + templates for requests.

# Phase 4: Interview Prep

# Behavioral Interview STAR Story Builder

I'm interviewing for: [Role] at [Company]
Job posting emphasizes: [e.g., "collaboration", "ownership", "learning from failure", "business acumen"]

Help me build 10 STAR stories:
1. Tell me a story about a time you: [dimension, e.g., "disagreed with a teammate"]
2. Context: [Situation, 1-2 sentences]
3. Challenge: [What was hard?]
4. Your action: [What did you do? Be specific.]
5. Result: [What happened? Do you have metrics?]

For each of the following dimensions, create a 2-minute STAR story:
- Conflict / disagreement with teammate or manager
- Ownership: owning a project end-to-end
- Learning from failure
- Leadership / stepping up
- Business acumen / finding an insight
- Collaboration across teams
- Handling ambiguity / adapting quickly
- Problem-solving (walking through your approach)
- Customer / user focus
- [1 additional story of your choice]

For each story, provide:
- Full story (2-minute version, ~350 words)
- 1-minute version (if asked to be brief, ~150 words)
- 3 ways to tie this back to [company's] values or the specific role
- Potential follow-up questions + answers
- Metrics or outcomes to emphasize

Format: 10 stories + timing guidance + tie-backs.

# Technical Interview Practice Questions

Role: [Your target role, e.g., "Data Engineer", "SWE", "Product Manager"]
Company interview style: [e.g., "Shopify is known for", "Google focuses on", "Startups emphasize"]

Generate 15-20 practice interview questions specific to [role]:
- Include mix of: depth questions (deep dive into one topic), breadth questions (testing across multiple areas), scenario-based questions
- For each question, provide:
  1. The question itself
  2. What the interviewer is really testing (skills / mindset)
  3. A strong answer framework (how to structure your response)
  4. 1-2 follow-up questions you might get
  5. Common mistakes to avoid
  6. How to demonstrate this skill in a real interview

Then provide:
- 3 recommended resources for deeper prep in each area
- A 4-week study plan (how many hours/week? What to focus on each week?)
- How to practice (mock interviews, platforms, etc.)

Format: Q&A guide + study plan + resources.

# System Design / Case Study Practice

Role: [Product Manager / Data Engineer / SWE, etc.]
Company: [Target company]

Give me 5 realistic case studies / design problems for [role]:
- Include 1-2 Canadian context problems (e.g., "Design a payment system for Shopify's Canadian merchants")
- Include mix of: business strategy, technical feasibility, metrics thinking

For each case study:
1. The problem statement
2. Follow-up questions you might get
3. A strong approach framework:
   - How to break down the problem
   - Key metrics to track
   - Trade-offs to consider
   - How to explain your thinking clearly
4. Sample answer (2-3 minutes, talking out loud)
5. What the interviewer is actually assessing
6. 3 ways to improve your answer (going deeper, showing creativity, considering edge cases)

Then provide:
- Weekly prep schedule (what to practice each week)
- How to practice out loud (talking-through your thinking, not just writing)
- Red flags in your answers (what sounds weak? How to strengthen?)

Format: 5 case studies + answer frameworks + practice guide.

# Interview Debrief & Feedback Template

Interview Details:
- Date, Role, Company, Interviewer Name
- Interview format (phone, video, in-person)
- Round # [1 of 3, etc.]

Questions Asked (list all):
1. [Q1]
2. [Q2]
... etc

For each question:
- Your answer: [Brief summary]
- Confidence level (1-5, where 5 is "nailed it")
- Why confident or not confident?
- What would you change about your answer?

Overall Assessment:
- Best moment in the interview (when did you feel strongest?)
- Weakest moment (when did you feel least confident?)
- Signals you picked up: Did they seem interested? Enthusiastic? Neutral?
- Did the conversation flow naturally, or did it feel scripted?

What You Learned:
- 3 things the interviewer emphasized (in questions or tone)
- 1-2 things you should prepare better for next interviews
- 1 thing you did well that you should repeat

Next Steps:
- Should you follow up? With what message?
- If rejected: Is this a good learning opportunity? What would you improve?
- If moving forward: What should you prep for round 2?

Format: Templated form + reflection questions.

# Phase 5: Offer & Negotiation

# Salary Benchmarking & Negotiation

I have an offer:
- Role: [Title]
- Company: [Name]
- Location: [City, Province]
- Base salary: [CAD $X]
- Bonus: [X%]
- Equity: [X shares / X% if startup]
- Other benefits: [remote, PTO, etc.]

My background:
- Years of experience: [X years]
- Education: [Degree, notable certifications]
- Current / comparable roles: [What have you made before?]

Help me:
1. Benchmark this offer:
   - Is it in-market for [role] in [city]? (Use Job Bank, Levels.fyi, Blind, Glassdoor, Comparably)
   - How does it compare to similar offers you've seen?
   - What's the typical range for someone with my background?
2. Is there room to negotiate? (Base salary, bonus, equity, benefits, remote flexibility, vacation, PD budget?)
3. Create a negotiation script:
   - Opening statement (how to open the conversation professionally)
   - Your ask (specific number or range)
   - Justification (based on market data + your value)
   - Alternative asks (if they can't move on base salary, can they move on equity, sign-on, etc.?)
   - How to respond if they say "no" or "this is our best offer"
   - How to finalize and close professionally
4. Total compensation comparison:
   - Current offer: [Total $ across all components, annualized]
   - Negotiated offer (projected): [X% higher, specific total]
   - Impact on your finances over 3 years
5. Non-salary negotiables (often easier to win):
   - Remote flexibility
   - Professional development budget
   - Flexible PTO
   - Mental health support
   - Stock vesting schedule
   - Sign-on bonus timing
6. Red flags in the offer (anything concerning?)

Format: Benchmark summary + negotiation script + total comp breakdown.

# Multi-Offer Comparison

I have multiple offers:

Offer A: [Company, Role, Base, Bonus, Equity, Location]
Offer B: [Company, Role, Base, Bonus, Equity, Location]
[Offer C, D, etc.]

Also consider:
- Your career goals (e.g., "Move into leadership in 2 years", "Build deep technical expertise", "Explore fintech")
- Your lifestyle priorities (e.g., "Want to stay in Canada", "Value remote work", "Need mentorship")
- Your growth trajectory (Where will each role take you in 3 years?)

Create:
1. Total compensation comparison (all offers, annualized over 3 years)
2. Career growth potential (where does each role lead?)
3. Learning opportunities (What new skills will you gain at each?)
4. Company stability + growth (trajectory of each company)
5. Team + culture (what do you know? What would you prioritize?)
6. Life / work balance (remote, commute, flexibility)
7. Scoring matrix:
   - Weight each factor (comp: 30%, growth: 25%, culture: 20%, stability: 15%, lifestyle: 10%)
   - Score each offer (1-10) on each factor
   - Calculate total weighted score
8. Gut feeling (What does your intuition say? Sometimes data doesn't capture everything.)
9. Final recommendation: Which offer aligns best with your goals?

Format: Comparison table + scoring matrix + gut check + final recommendation.

# Post-Rejection & Feedback Loop

I interviewed at: [Company, Role, Round #]
Outcome: [Rejected / Not moving forward]

Help me convert this into a learning opportunity:
1. What I learned about the role + company from the interview
2. Where I felt strongest in the interview
3. Where I felt weakest (be honest)
4. Follow-up email:
   - Thank them professionally
   - Express genuine interest in the role / company
   - Ask for 1-2 specific pieces of feedback
   - Leave door open for future opportunities
   - Professional tone + specific details (vs generic "thanks anyway" email)
5. Lessons for future interviews:
   - What should I prepare better for next time?
   - What question surprised me? How should I prepare for similar questions?
   - What would I do differently?
6. Should I apply to other roles at this company? (When? Which teams?)
7. How to stay on their radar (without being annoying) – ideas for follow-up in 6-12 months

Format: Reflection guide + follow-up email template + lessons learned + next steps.
